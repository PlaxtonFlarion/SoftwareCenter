## 🔐 使用通行证激活

软件在未激活状态下可正常启动，但**核心功能将被锁定**。用户可通过输入通行证进行激活，以解锁完整功能。

---

### 激活流程

1. 在终端中执行激活命令并输入通行证（格式如：`XXXX-XXXX-XXXX`）
2. 系统将自动联网验证并将通行证绑定到当前设备
3. 激活成功后将在本地生成 `.lic` 文件，用于后续离线校验

```
framix --apply Framix-XXXX-XXXX-XXXX
```

```
memrix --apply Memrix-XXXX-XXXX-XXXX
```

---

### 授权机制说明

- 每个通行证最多可绑定 **3 台设备**
- 激活后会生成 `.lic` 文件，支持 **离线运行**
- 启动时系统将尝试 **每日** 联网检查授权状态
- 无需解绑设备，若授权失效可直接使用新的通行证重新激活

### .lic 授权文件说明

激活成功后，系统将在本地生成一个授权文件（如：`framix_signature.lic` 或 `memrix_signature.lic`），其内容采用 Base64 编码，并由签名保护，不可伪造或篡改。

授权结构示例（解码后的 JSON 格式）：

```
{
  "app": "XXXXXX",
  "code": "XXXXXX-Key***************************************",
  "castle": "6027578db1******************************************************",
  "expire": "2025-12-31",
  "issued": "2025-05-25T07:29:57.199864+00:00",
  "issued_at": "2025-05-25T07:12:36.436505+00:00",
  "license_id": "64f6c0d790******************************************************",
  "interval": 86400
}
```

字段说明：

- `app`：应用标识，用于识别授权来源（如 Framix、Memrix）
- `code`：激活码（通行证）本体  
- `castle`：绑定的设备唯一标识符（通常为指纹哈希）  
- `expire`：该激活码的有效截止时间（UTC，仅日期）  
- `issued`：本次授权验证时间（UTC 时间戳）  
- `issued_at`：授权首次签发时间（UTC 时间戳）    
- `license_id`：授权记录唯一编号（可用于后台查询与审计）  
- `interval`：最小授权检查间隔（单位：秒，86400 表示每日一次）

授权文件仅可由官方授权服务器签发，使用 RSA 签名机制保护。每次启动时将验证该文件合法性和有效性。

## ⚠️ Windows 安全提示说明：无数字签名

在首次运行某些从浏览器下载的 `.exe` 可执行文件时，Windows 可能会弹出如下安全警告：

> **“无法验证发布者”**  
> **“此文件没有包含有效的数字签名。”**

### 原因说明

该提示出现是因为该程序未使用受信任的数字签名证书进行签名。  
Windows 出于安全考虑，对所有未知来源的可执行文件都会提示此类信息。

### 如何继续运行？

1. 当弹出提示时，点击 **“详细信息”**；
2. 然后点击 **“仍要运行”** 或 **“运行”**；
3. 如果文件运行被阻止，请：
   - 右键点击该文件，选择 **“属性”**；
   - 在底部勾选 **“解除锁定”**；
   - 点击“应用”后重新运行即可。

### 安全性声明

- 程序文件未包含恶意行为，仅根据用户授权执行；
- 所有可执行文件均可通过 SHA256 等方式验证其完整性；
- 程序不会进行自动安装或系统更改操作，除非用户授权。

### 后续计划

如果该软件未来发布到更多平台，可能会引入数字签名证书以减少此类提示。当前版本仅供已知用户使用，系统提示为正常现象。

## ⚡️ 快速开始（macOS）

### 安装

打开对应软件的 `.dmg` 安装包，按提示将图标拖入「应用程序」或任意目录。

示例：

- Framix → 拖入 `/Applications/Framix.app`
- Memrix → 拖入 `/Applications/Memrix.app`

---

### 添加环境变量

如需在终端中直接运行命令，可将二进制路径添加到系统 `PATH`：

```
# Framix 示例
echo 'export PATH="/Applications/Framix.app/Contents/MacOS:$PATH"' >> ~/.zshrc

# Memrix 示例
echo 'export PATH="/Applications/Memrix.app/Contents/MacOS:$PATH"' >> ~/.zshrc

source ~/.zshrc
```

---

### 授权启动脚本（首次运行前）

```
# Framix
chmod +x /Applications/Framix.app/Contents/MacOS/framix
chmod +x /Applications/Framix.app/Contents/MacOS/framix.sh

# Memrix
chmod +x /Applications/Memrix.app/Contents/MacOS/memrix
chmod +x /Applications/Memrix.app/Contents/MacOS/memrix.sh
```

## 🛡 macOS 安全提示说明（首次启动）

由于本应用未通过 App Store 分发，macOS 的安全机制（Gatekeeper）**可能在首次启动时弹出提示**，显示“无法验证开发者”或“已被阻止使用”。

如遇此情况，请按以下步骤操作以启用应用：

1. 打开 **系统设置** > **隐私与安全性**
2. 滚动到页面底部，找到“允许从以下位置下载的应用”部分
3. 点击“仍要打开”或“允许来自未知开发者的应用”
4. 再次双击 `.app` 图标即可正常启动

此操作仅需执行一次，后续启动不会再提示。

> 此行为是 macOS 的默认安全策略，不影响应用功能或安全性。
